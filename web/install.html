Installing AVR-Ada
   


Unpackage

Extract all files from the package AVR-Ada-V0.5 at a directory that suits you.

tar xvfj avr-ada-0.5.2.tar.bz2


Build the run time system

Just type make in the installation directory. This will call ./configure and compile the run time system (RTS) and AVR library files as necessary. Please tell us, if it does not!

cd AVR-Ada-V0.5
make

After compiling the RTS, the make command copies the necessary files of the RTS into your compiler's installation directory. The AVR files will stay in the local avr directory for now. Future versions of AVR-Ada will probably install the AVR files together with the RTS.

The AVR library contains some support packages and the necessary definition files for the part specific constants. Constants are always inlined in GNAT and thus no corresponding object files are needed. The provided gpr file (avr/avr.gpr) compiles the support packages. You have to verify the part description package for your actual processor.


Verify installation

Verify the installation by compiling the example programs in the apps directory. A simple make samples from the top level directory should be sufficient to build all sample applications.

Please tell us about any success or failure!
